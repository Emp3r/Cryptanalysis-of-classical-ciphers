Caesar cipher

Summary

One of the simplest substitution ciphers. Each letter in the plaintext is replaced by a letter shifted by number of positions down the alphabet. Key is single character from english alphabet.

History

The Caesar cipher is named after Julius Caesar, who used it (with a shift of three) to protect military messages back in the 1st century BC. It was probably effective, since the first frequency analysis records date to the 9th century AD in the Arab world. There is evidence that Julius Caesar used more complicated systems as well.
A Caesar cipher with a shift of one is also used on the back of the Hebrew mezuzah (verses from the Torah written on animal skin) to encrypt the names of God.

Algorithm


Encryption is performed by shifting every letter from plain text by fixed number of positions (key) and rewriting the letter by assigned substitution. Key can be any single character from alphabet (a-z). As shown in the picture, key 'c' shifts text by 2 positions, so every letter 'a' in plain text is rewriten as 'b' in coded text and so on for other letters. Key 'a' means the text shifts by 0 positions, so the text doesn't encrypt at all. For 'z' the text is shifted by 25 positions (also -1).

Examples

1. example

Plain text: "Carpe diem"
Key: "F"
Coded text: "HFWUJ INJR"

2. example

Plain text: "The only way of finding the limits of the possible is by going beyond them into the impossible"
Key: "R"
Coded text: "KYV FECP NRP FW WZEUZEX KYV CZDZKJ FW KYV GFJJZSCV ZJ SP XFZEX SVPFEU KYVD ZEKF KYV ZDGFJJZSCV"

Advantages

Encrypting algorithm is quick and easy to understand. Anyone can encrypt or decrypt messages in a few minutes with only pen and paper.

Weaknesses

Caesar cipher has only 26 different keys (number of letters in alphabet). It's very easy to try all of them and tell which one is correct. Beside that key 'a' leaves the plain text unchanged.

Cracking

Brute force - Frequency analysis

Algorithm decrypts coded text with all 26 keys and calculates which has the most similar letter frequency table with expected plain text frequency. Returns key with the smallest difference.

Brute force - Real words count

Algorithm decrypts coded text with all 26 keys and calculates which one counts the most real words. Real words are defined by top used words in chosen language. Returns key with the largest count.
